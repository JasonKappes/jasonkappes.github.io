KAP Engine

Kap Engine is an independent 6 month software development project focused on creating a 3D video game engine in C++.
It was built on top of a basic OpenGL graphics engine. I focused on robust, scalable, and optimized system design
that prioritized de-coupling. Most systems utilize design patterns such as singleton, strategy, visitor, and attorney/client.

Gameplay Showcase

This video showcases the collaboration of various gameplay systems
inside my engine. This gameplay more or less acted as an engine
"stress test" and is a gage for the type of games that can be
created using Kap Engine.

World Editor Showcase

This video showcases the various capabilities granted to a user
in world editor mode. Any 'scene' that the user creates can be ran
with world edit mode enabled. This allows the user to select any
object in D space and change its world data, duplicate, or delete
the selected object. Any changes made for an object in world edit
mode are saved.

Kap Engine Systems Showcase

| GameObject: The user can inherit from the GameObject class and thus utilize many engine-level capabilities.
| 3D Object Collision: This supports collisions between 3D objects. The user can select a Bsphere, OBB, or AABB collision model for any object.
| 3D Mouse Selection: This allows the user to detect collisions between mouse and a 3D object.
| Terrain Management: Terrain generation and customization from user files and efficient terrain collision.
| Asset Management: Texture, Model, and Shader file management.
| Rendering: Objects can be rendered as 2D or 3D objects. A basic graphics engine was used for shading, texturing, and modeling of 3D objects.
| Alarm: The user can receive function callbacks after a defined amount of time.
| World Editor: The world can be seen and world data (position/rotation/scale) changed for 3D objects in realtime.
| Serialization: Gameobject world data can be changed in world edit mode and updated upon a regular game execution.
